HTML + SASS + Vanilla JS + Bootstrap 4

Tebrikler örnek uygulamayı başarıyla çalıştırdınız.

Çalışma ortamını ayağa kaldırabilmek için NodeJS ve NPM yüklü olmalıdır. Benim bilgisayarımda Node v14.15.3 ve NPM 6.14.9 versiyonları bulunmaktadır. NodJS kurulumu zaten otomatik olarak NPM kurulumu da yapmaktadır. Kurulumu işlemlerini tamamlandıktan sonra npm install komutunu kullanarak package.json içerisinde bulunan tüm paketleri yüklemelisiniz. Bu komut zaten tek seferde ilgili bağımlılıkları yükleyecektir.

Üç farklı npm script bulunmaktadır. Package.json dosyasında bu scriptleri inceleyebilirsiniz. Ayrıca yapılandırma dosyalarına da bakmanızı tavsiye ediyorum.

  1. npm start komutu ile dist klasörü altında herhangi bir dosya oluşturmadan arkaplanda index.js dosyası ve ilgili yapılandırma dosyaları temel alınarak build sürecinden geçirilerek uygulama varsayılan tarayıcıda ayağa kaldırılır. Bu esnada aslında development sürecinde uygulama çalıştırıldığı için source maps dosyaları kullanılabilir durumda olacaktır. Bu sayede hata ayıklamak kolay olacaktır. Hatanın hangi dosyada olduğu tarayıcı üzerinde bulunabilecektir.
  2. npm run build-dev komutu ile uygulama baştan sonra derlenerek dist klasörü altına sıkıştırılmadan source maps dosyaları ile birlikte çıkartılır ancak uygulama çalıştırılmaz.
  3. npm run build-prod komutu ile uygulama baştan sonra derlenerek dist klasörü altına sıkıştırılarak source maps dosyaları oluşturulmadan çıkartılır ve uygulama çalıştırılmaz.

Bootstrap dosyası node_module altında bulunmaktadır. Projeye dahil olduğu yer main.scss dosyasıdır. Bu dosyaya da vendors/_bootstrap.scssyolunu takip ederek ulaşabilirsiniz.

Ana dizinde bulunan .browserlistrc dosyası Bootstrap için hazırlanan dosyanın aynısıdır. Bootstrap temeli olduğu için destek verdikleri tarayıcılara göre hareket etmek doğru bir yaklaşım olacaktır.

Proje derlemesi yapılırken index.js kontrol edilmektedir. Burada tüm scss dosyalarının toplanmış olduğu main.scss dahil edilmiştir. Bu dosya içerisindeki tüm bağlı scss dosyaları derlenerek prefixleri ile dist altına çıkartılmaktadır.

Pages altında oluşturacağınız tüm dosyalara otomatik olarak javascript ve css dosyaları dahil edilmektedir. Yeni bir sayfa eklerken webpack.common.js üzerinde bulunan htmlPageNames isimli diziye oluşturmuş olduğunuz html sayfasının adını yazınız.

Bootstrap 4 javascript dosyası direkt olarak index.js üzerinde dahil edilmiştir. Ayrıca JQuery bağımlılığıda olduğu için bu dosyada npm ile node_modules altına yüklenip çağrılmıştır. Ancak slim versiyonunun eklendiğini unutmayın. HTTP işlemleri için JQuery kullanacak olursanız özelliklerinden faydalanamazsınız. Bunun için tam özellikli dosyayı dahil etmelisiniz. Ancak saf javascript yazacaksanız problem olmayacaktır.

Kendinize ait (özel) Javascript kodlarınızı src/js yolu altında oluşturup app.js dosyasına dahil edebilirsiniz. App.js dosyasıda index.js dosyasında olduğu için otomatik olarak tüm js dosyaları babel aracılığıyla derlenip ES5(ES2009) versiyonuna çevrilerek sayfalarınıza dahil edilecektir.

Bootstrap özellikleri veya biçimlendirmeleri yeterli gelmezse kendi css kodlarınızı scss klasörü altında bulunan dosyalarda oluşturabilirsiniz. Düz CSS kodu yazmış olsanızda problem olmayacaktır. Ancak SASS(SCSS) öğrenmeniz yararınıza olacaktır. Bu dosyaların tamamı tek bir noktadan yönetilmektedir. Bu dosya main.scss dosyasıdır. Buraya dahil edilmeyen hiçbir kod derlenmez.

SCSS klasörünü istediğiniz gibi özelleştirebilirsiniz. Ancak main.scss dosyasında ilgili değişiklikleri, ekleme, çıkarma işlemlerini gerçekleştirmelisiniz.

Sonuç olarak tüm bu yapı tek bir Javascript(ES5) ve tek bir CSS(POSTCSS-Autoprefixer) dosyası haline dönüştürlecektir. Ayrıca statik sayfaların herbirine bu dosyalar otomatik olarak enjekte edilecektir. Aynı şekilde örnek olarak sizlere bırakmış olduğum bir img etiketi ile görsel ekleme işlemi ve dark-theme.scss içindeki arkaplana svg dosyası ekleme işlemindeki görsel unsurlarda dist/images klasörü altında yerlerini alacaklardır. Bu dosyaların bağlantılarıda otomatik olarak güncellenecektir.
